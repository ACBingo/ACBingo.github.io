LeetCode_5:Longest Palindromic Substring

  • LeetCode
  • String

find the longest palindromic substring in S.
讨论了O(n2)的朴素算法,O(nlogn)的kmp扩展算法,O(n)的Manacher算法

题目

Longest Palindromic Substring

参考

例外的，这篇文章先给出参考，leetcode某位国外大神写的两篇很详细的文章,质量相当的高啊(比国内博客好的不是一点半点= =):
Longest Palindromic Substring Part I
Longest Palindromic Substring Part II

文章比较长，但是强烈建议仔细读一下.

暴力

O(n3)
暴力找，我就不想再多说了= =

中心扩展算法

枚举中点，然后向两边扩展，时间复杂度是O(n2)

但是有一点需要注意的是，回文的情况可能是 a b a，也可能是 a b b a。
还得分奇偶性分别判断,恶心死了

可以通过在两个字符之间插入一个#，然后让整个字符串变为奇数。
即:aba -> #a#b#a#
abba -> #a#b#b#a#
这样处理起来不就统一啦,一番分析后发现,最后的真实长度就是(L-1)/2。

代码就不写了.

DP

时间和空间复杂度都为O(n2)
定义p[i,j],若Si..Sj是回文串,为true否则为false
那么转移方程就是:P[ i, j ] ← ( P[ i+1, j-1 ] and Si = Sj )

初始值为:
P[ i, i ] ← true
P[ i, i+1 ] ← ( Si = Si+1 )

代码

直接贴参考文章中的代码吧，写的挺好的

后缀树

时间复杂度为O(N*logN)
比较复杂..如果你有兴趣的话:(Longest Palindromic Substring)(https://en.wikipedia.org/wiki/Longest_palindromic_substring)

KMP扩展

时间复杂度为O(N*logN).
KMP算法一直是我头疼的地方，每次都是看懂了后过段时间就忘= =，所以每次碰到字符串处理的问题我就害怕。。

不过求一个简简单单的回文串，还得动用KMP?下面有更优的算法

Manacher算法

正如原文中所说:

Does an O(N) solution exist? You bet! However, it is not trivial and requires some very clever observation.

这个算法一般人真是看不出来啊= =

原文讲的很清楚了:Manacher算法,我就不啰嗦了
